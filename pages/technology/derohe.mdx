## Homomorphic Encryption
Homomorphic encryption is a game-changer for data security, enabling calculations on encrypted data while keeping it private. It's pivotal for secure cloud computing, compliance with regulations, and fostering secure data collaboration, ensuring both privacy and enhanced security.

Dero merges homomorphic encryption with an account-based model, ensuring instant transactions, robust security, and privacy. Σ mining and Σ blocks employ a network-wide consensus, rewarding all miners and confirming transactions instantly within the same block.

What sets DERO apart is its encrypted blockchain and account-based model, offering true privacy on-chain. This unique system not only ensures security but also allows users to check balances without syncing the entire chain, making balance inquiries homomorphically encrypted and easily accessible.

Homomorphic encryption enables operations on encrypted data without decryption, preserving data confidentiality. DERO's encrypted blockchain, coupled with this encryption, guarantees top-notch privacy, security, instant transactions, settlement, and balance inquiries.

## DHEBP
The DERO Homomorphic Encryption Blockchain Protocol (DHEBP) demonstrates Fully Homomorphic Encryption (FHE) in blockchain transactions. Each transaction consists of two main parts:

1. **Statement:** Contains ring member keys, commitments, and encrypted balances. The size grows linearly and consists of 4 curve points per ring member.
2. **Proof:** This part's size grows logarithmically concerning ring members. For example, a proof for an 8-ring size transaction is 1997 bytes, and for a 512-ring size, it's 3965 bytes.

The build process involves switching to the directory containing the readme file, setting the GOPATH, and running the code using Go (version 1.12.7). The output sample illustrates the creation and successful execution of transactions, showcasing the transfer of funds between sender and receiver accounts, along with the changes in account balances.

DHEBP stands out for its decentralization, privacy, and scalability. Wallets in DERO can execute one spending transaction at a time, preventing double spending and allowing immediate subsequent transactions settled within the same block for instant confirmation.
