# Gnomon
![Gnomon](/assets/gnomon.jpeg)
## Decentralized Search Engine
DeroHE's client, powered by Gnomon, empowers individual user nodes by enabling direct navigation, exploration, and search within the blockchain and its smart contracts. This liberates users from relying on external blockchain data providers like Infura, offering more control over data access and management. Gnomon efficiently handles and arranges the gathered information, thus enhancing its operational efficiency within the decentralized ecosystem.

Gnomon works effectively with databases like BoltDB and GravitonDB. It mainly stores, indexes, and fetches data from the DERO blockchain. These databases help organize and manage blockchain information, making it easier to search and use in decentralized apps. BoltDB and GravitonDB play a crucial role in storing indexed blockchain data, boosting Gnomon's ability to search and organize information for various decentralized services.

## BoltDB
BoltDB is a key-value pair embedded database that Gnomon utilizes for storage.

#### Setup Process
**Initialization:** Gnomon initializes BoltDB to store and manage data efficiently.
**Database Creation:** It creates or opens a BoltDB database file, establishing a storage location for indexing.

#### Functions and Operations
**Data Retrieval:** Gnomon retrieves indexed data using various functions provided by BoltDB. For instance:
  - Retrieving validated Smart Contract IDs (SCIDs) and their respective owners.
  - Getting details of SC transfer transactions, SCID invokes, etc.
**Last Indexed Height:** It keeps track of the last indexed block height for continuing data indexing.

#### Example Use Cases
- Gnomon retrieves information such as SCIDs, transaction details, and SC-specific data stored in BoltDB.
- It manages database operations, including storing and querying indexed data for various purposes, such as Smart Contract interaction details and transaction history.

## GravitonDB
GravitonDB is another database option used by Gnomon, mainly focused on enhancing performance and scalability.

#### Initialization
**Import:** Gnomon imports the GravitonDB library/module to use it as an alternative database option.
**Database Setup:** It sets up GravitonDB by initializing and specifying the database type (GravitonDB) for storage.

#### Operations and Use Cases
- **Similar Operations:** Gnomon performs similar operations in GravitonDB as it does in BoltDB, such as storing and retrieving indexed data, managing the last indexed height, and fetching Smart Contract-related information.

#### Potential Advantages
- GravitonDB may offer optimizations for specific use cases, potentially enhancing speed and efficiency in handling indexed blockchain data.
- It might provide different functionalities or optimizations compared to BoltDB, catering to diverse database requirements.
