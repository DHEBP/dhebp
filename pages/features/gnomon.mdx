## Indexing Blockchain Data
Gnomon interacts with databases primarily for indexing and storing blockchain-related data obtained from the DERO blockchain. The databases used include BoltDB and GravitonDB, each serving specific purposes within the Gnomon ecosystem.

Gnomon's interaction with databases like BoltDB and GravitonDB involves managing storage, indexing blockchain-related data, and retrieving information necessary for decentralized search functionality. These databases play a crucial role in storing and organizing indexed data retrieved from the DERO blockchain, allowing Gnomon to efficiently query and index information for various decentralized applications and services.

## BoltDB
BoltDB is a key-value pair embedded database that Gnomon utilizes for storage.

#### Setup Process
1. **Initialization:** Gnomon initializes BoltDB to store and manage data efficiently.
2. **Database Creation:** It creates or opens a BoltDB database file, establishing a storage location for indexing.

#### Functions and Operations
- **Data Retrieval:** Gnomon retrieves indexed data using various functions provided by BoltDB. For instance:
  - Retrieving validated Smart Contract IDs (SCIDs) and their respective owners.
  - Getting details of SC transfer transactions, SCID invokes, etc.
- **Last Indexed Height:** It keeps track of the last indexed block height for continuing data indexing.

#### Example Use Cases
- Gnomon retrieves information such as SCIDs, transaction details, and SC-specific data stored in BoltDB.
- It manages database operations, including storing and querying indexed data for various purposes, such as Smart Contract interaction details and transaction history.
